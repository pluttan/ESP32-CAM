# ESP32-CAM

**Сканер QR-кодов с живой трансляцией с камеры на ESP32-CAM**

</div>

Сканер QR-кодов на модуле AI-Thinker ESP32-CAM. Камера захватывает кадры JPEG 640x480, обрабатывает их через библиотеку quirc для обнаружения QR в реальном времени и предоставляет встроенный веб-интерфейс — живой MJPEG-поток и декодированный текст QR — через собственную точку доступа WiFi.

## ■ Возможности

- ❖ **Сканирование QR в реальном времени** — кадры JPEG декодируются в оттенки серого и обрабатываются через quirc каждые 300мс при разрешении 640x480
- ❖ **WiFi AP** — автономная точка доступа WPA2 (`ESP32-CAM`, пароль `12345678`), маршрутизатор не нужен
- ❖ **Веб-интерфейс** — встроенная HTML-страница, опрашивающая декодированный текст через JSON-endpoint `/qr`
- ❖ **Живой MJPEG-поток** — отдельный HTTP-сервер на порту 81 (`/stream`), чтобы трансляция не блокировала `/qr`
- ❖ **Удержание 2 секунды** — обнаруженный QR-результат сохраняется 2с для предотвращения мерцания
- ❖ **Нативный ESP-IDF** — построен на фреймворке esp-idf (не Arduino) для полного аппаратного контроля
- ❖ **Потокобезопасность** — буфер QR-результата защищён мьютексом для конкурентного HTTP-доступа
- ❖ **PSRAM** — буфер RGB-кадра и структуры quirc размещены в PSRAM

## ■ Стек

<div align="center">

| Компонент | Технология |
|-----------|------------|
| МКУ | ESP32 (AI-Thinker ESP32-CAM, PSRAM) |
| Камера | OV2640 @ 640x480 JPEG (esp32-camera) |
| Фреймворк | ESP-IDF 5.5 |
| QR-библиотека | quirc |
| Сборка | PlatformIO / CMake |
| HTTP | esp_http_server (port 80 page+`/qr`, port 81 `/stream`) |

## ■ Как это работает

```
1. Устройство загружается и создаёт точку доступа WPA2 (SSID: ESP32-CAM)
2. OV2640 захватывает кадры JPEG 640x480 каждые 300мс
3. Кадры преобразуются в оттенки серого и передаются в QR-декодер quirc
4. Декодированный QR-текст сохраняется в защищённый мьютексом буфер PSRAM и удерживается 2 секунды
5. Порт 80 отдаёт веб-страницу и JSON-endpoint /qr для опроса декодированного текста
6. Порт 81 независимо транслирует MJPEG-видео, чтобы сканирование не блокировало живой поток
```

## ■ Использование

```bash
# Build and upload
pio run -t upload

# Monitor
pio device monitor
```

1. Включите модуль
2. Подключитесь к WiFi `ESP32-CAM` (пароль: `12345678`)
3. Откройте `http://192.168.4.1` в браузере — страница показывает живой поток (порт 81) и декодированный текст
4. Наведите камеру на QR-код
